# AOS — System Discovery Conversation

## Purpose

Before any system is generated, AOS must first understand what is being created.

Generation without understanding produces artifacts.

Conversation produces architecture.

Therefore:

> Conversation precedes generation.

The purpose of the System Discovery Conversation is to transform an idea into a structured system definition.

---

# Phase 1 — Wisdom

Before discussing implementation:

Should this exist?

Questions:

- What problem is being solved?
- Who benefits?
- What harm could result?
- Why should this exist?
- Why now?
- Is there a simpler solution?

Only after wisdom approves creation does the conversation continue.

---

# Phase 2 — Identity

What is this?

Examples:

- Product
- Service
- Software
- Website
- Business
- Educational course
- Operating system
- Marketplace
- Community
- Internal tool
- Other

---

# Phase 3 — Purpose

Describe the system in one sentence.

This purpose statement becomes the foundation for every future decision.

---

# Phase 4 — Users

Who uses this system?

Possible roles include:

- Customer
- Administrator
- Designer
- Operator
- Salesperson
- Vendor
- AI Assistant

Each role receives different permissions.

---

# Phase 5 — Objects

What things exist inside the system?

Objects become the nouns of the system.

---

# Phase 6 — Workflows

What processes occur?

Workflows become the verbs of the system.

---

# Phase 7 — Templates

What repeats?

Anything repeated should become a reusable template.

---

# Phase 8 — Structure

Separate structure from content.

Reusable:

- Layout
- Rules
- Workflow
- Logic

Replaceable:

- Text
- Images
- Pricing
- Locations

---

# Phase 9 — Automation

Which tasks belong to humans?

Which belong to AI?

Which should be automatic?

Automation should preserve human authority.

---

# Phase 10 — Outputs

What should this system generate?

Examples:

- Websites
- Apps
- Documentation
- SOPs
- Marketing
- PDFs
- Images
- APIs

Outputs are generated only after the previous phases are complete.

---

# Completion

When the conversation is complete, AOS produces a System Definition.

That definition becomes the single source of truth from which websites, software, documentation, marketing, and future improvements are generated.
